## 文件结构
- cpp:包含用c++20实现的AST到RISCV汇编指令的项目代码。
- toyc-interpreter:包含用ocaml实现的源代码到AST的项目代码。
- center.cpp:项目的链接程序，可以将输入转为tc文件，并调用相应的处理程序进行处理。
- README.md:包含对项目的介绍。
- Makefile:项目构建脚本。

## 使用方法
Makefile设定会识别操作系统类型而执行对应的脚本，目前在Linux下运行正常。
在根目录下执行`make build`可以自动构建项目，并在当前路径生成对应的链接程序，前端翻译器和后端翻译器。
在执行完make build后，可以执行编译器主程序`compiler`。注意，不同模块因实现不同，调用方法不同：
- compiler：编译器链接模块，从标准输入流获取信息，输出到标准输出流
- front：前端模块，输入文件名，输出到标准输出流。
- back：后端模块，从标准输入流获取信息，输出到标准输出流。
在根目录下执行`make clean`可以清除生成的项目文件。
